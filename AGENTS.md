# OMAR-TOP

A TUI resource monitor for Omarchy Linux systems. Uses the same theme-injection mechanism as btop ‚Äî colors follow Omarchy themes automatically.

## Quick Start

```bash
omar_top                    # launch
q / ctrl+c / esc            # quit
omarchy theme set "Tokyo Night"  # colors update live
```

Binary: `~/.local/bin/omar_top`
Config: `~/.config/omar_top/config.toml`

## Architecture

Go + [Bubble Tea](https://github.com/charmbracelet/bubbletea) (Elm-architecture TUI framework) + [Lip Gloss](https://github.com/charmbracelet/lipgloss) (CSS-like terminal styling).

## Theme Integration

Replicates btop's mechanism exactly:

| Step | What |
|------|------|
| Template | `~/.config/omarchy/themed/omar_top.theme.tpl` ‚Äî maps `colors.toml` vars to omar_top's color slots |
| Rendered | `~/.config/omarchy/current/theme/omar_top.theme` ‚Äî auto-generated by `omarchy-theme-set-templates` |
| Symlink | `~/.config/omar_top/themes/current.theme` ‚Üí `.../current/theme/omar_top.theme` |
| Config | `color_theme = "current"` in `~/.config/omar_top/config.toml` (default) |
| Reload | Polls theme file mtime every tick + handles SIGUSR2 |

### Color slots (template)

```ini
theme[bg]        ‚Äî full-screen background
theme[fg]        ‚Äî default foreground
theme[title]     ‚Äî "OMAR-TOP" header
theme[accent]    ‚Äî highlight accent
theme[bar_bg]    ‚Äî progress bar empty portion
theme[bar_fill]  ‚Äî progress bar fill
theme[label]     ‚Äî section labels (CPU, MEM, etc.)
theme[value]     ‚Äî numeric values (percentages, speeds)
theme[divider]   ‚Äî horizontal divider line
theme[info]      ‚Äî secondary info (temp, freq, etc.)
```

Default fallback (if no theme file exists) is a Catppuccin-inspired dark palette.

## Metrics

All collectors read from the Linux `/proc` and `/sys` filesystems ‚Äî zero external dependencies. Updated every tick (default 1s).

### CPU
- `/proc/stat` ‚Äî delta of `cpu` line: `(total - idle) / total * 100`
- `/proc/cpuinfo` or `scaling_cur_freq` ‚Äî frequency
- `/sys/class/thermal/thermal_zone0/temp` ‚Äî temperature in ¬∞C

### GPU
- **NVIDIA**: `nvidia-smi --query-gpu=utilization.gpu,temperature.gpu,memory.used,memory.total`
- **AMD/Intel**: `/sys/class/drm/card*/device/gpu_busy_percent` + hwmon temp via driver name match
- Section auto-hides if no GPU detected or no driver data

### Memory
- `/proc/meminfo` ‚Üí MemTotal, MemAvailable; `Used = Total - Available`

### Disk
- `/proc/mounts` filtered to physical filesystems (ext4, btrfs, xfs, etc.)
- `syscall.Statfs` for each mountpoint ‚Üí used/total bytes
- Shows root `/` by default

### Network
- `/proc/net/dev` ‚Äî RX/TX byte counters per interface
- Tracks deltas across ticks for bytes/s rate
- Auto-selects interface with highest traffic, skips loopback
- Speeds displayed in decimal units (kB/s, MB/s, GB/s)

## Config (`~/.config/omar_top/config.toml`)

```toml
color_theme = "current"       # theme file name (without .theme)
update_ms = 1000              # refresh interval in ms (min 100)
theme_background = true       # fill background with theme bg color
```

## Dependencies

Runtime: none (reads from procfs/sysfs).
Build: Go 1.23+, `github.com/charmbracelet/bubbletea v1.3.10`, `github.com/charmbracelet/lipgloss v1.1.0`.
