# OMAR-TOP

A TUI resource monitor for Omarchy Linux systems. Uses the same theme-injection mechanism as btop — colors follow Omarchy themes automatically.

## Quick Start

```bash
omar_top                    # launch
q / ctrl+c / esc            # quit
omarchy theme set "Tokyo Night"  # colors update live
```

Binary: `~/.local/bin/omar_top`
Config: `~/.config/omar_top/config.toml`

## Architecture

Go + [Bubble Tea](https://github.com/charmbracelet/bubbletea) (Elm-architecture TUI framework) + [Lip Gloss](https://github.com/charmbracelet/lipgloss) (CSS-like terminal styling).

### File layout

```
omar_top/
├── main.go       Entry point, SIGUSR2 handler
├── go.mod / go.sum
├── AGENTS.md     This file
├── config.go     Reads ~/.config/omar_top/config.toml (manual TOML parse, no dependency)
├── theme.go      Parses theme[key]="#hex" files, exposes a Theme struct for Lip Gloss
├── model.go      Bubble Tea model: state, metrics struct, ticker, delta collectors
├── update.go     Message dispatch: tick, resize, keypress, theme reload
├── view.go       UI rendering: header, divider, 5 metric sections
├── format.go     Human-readable formatting: binary (KiB/GiB) for storage, decimal (kB/s, MB/s, GB/s) for network, freq, temp, percent
├── format_test.go Unit tests for format functions
├── cpu.go        Reads /proc/stat + /sys/devices/system/cpu/ + thermal
├── gpu.go        Auto-detects NVIDIA/AMD/Intel; uses nvidia-smi or sysfs
├── memory.go     Reads /proc/meminfo
├── disk.go       syscall.Statfs on physical mountpoints from /proc/mounts
└── network.go    Reads /proc/net/dev, tracks RX/TX delta for rate
```

## Theme Integration

Replicates btop's mechanism exactly:

| Step | What |
|------|------|
| Template | `~/.config/omarchy/themed/omar_top.theme.tpl` — maps `colors.toml` vars to omar_top's color slots |
| Rendered | `~/.config/omarchy/current/theme/omar_top.theme` — auto-generated by `omarchy-theme-set-templates` |
| Symlink | `~/.config/omar_top/themes/current.theme` → `.../current/theme/omar_top.theme` |
| Config | `color_theme = "current"` in `~/.config/omar_top/config.toml` (default) |
| Reload | Polls theme file mtime every tick + handles SIGUSR2 |

### Color slots (template)

```ini
theme[bg]        — full-screen background
theme[fg]        — default foreground
theme[title]     — "OMAR-TOP" header
theme[accent]    — highlight accent
theme[bar_bg]    — progress bar empty portion
theme[bar_fill]  — progress bar fill
theme[label]     — section labels (CPU, MEM, etc.)
theme[value]     — numeric values (percentages, speeds)
theme[divider]   — horizontal divider line
theme[info]      — secondary info (temp, freq, etc.)
```

Default fallback (if no theme file exists) is a Catppuccin-inspired dark palette.

## Metrics

All collectors read from the Linux `/proc` and `/sys` filesystems — zero external dependencies. Updated every tick (default 1s).

### CPU
- `/proc/stat` — delta of `cpu` line: `(total - idle) / total * 100`
- `/proc/cpuinfo` or `scaling_cur_freq` — frequency
- `/sys/class/thermal/thermal_zone0/temp` — temperature in °C

### GPU
- **NVIDIA**: `nvidia-smi --query-gpu=utilization.gpu,temperature.gpu,memory.used,memory.total`
- **AMD/Intel**: `/sys/class/drm/card*/device/gpu_busy_percent` + hwmon temp via driver name match
- Section auto-hides if no GPU detected or no driver data

### Memory
- `/proc/meminfo` → MemTotal, MemAvailable; `Used = Total - Available`

### Disk
- `/proc/mounts` filtered to physical filesystems (ext4, btrfs, xfs, etc.)
- `syscall.Statfs` for each mountpoint → used/total bytes
- Shows root `/` by default

### Network
- `/proc/net/dev` — RX/TX byte counters per interface
- Tracks deltas across ticks for bytes/s rate
- Handles 64-bit counter wraparound (skips interface on that tick)
- Auto-selects interface with highest traffic, skips loopback
- Speeds displayed in decimal units (kB/s, MB/s, GB/s, up to PB/s)

## Config (`~/.config/omar_top/config.toml`)

```toml
color_theme = "current"       # theme file name (without .theme)
update_ms = 1000              # refresh interval in ms (min 100)
theme_background = true       # fill background with theme bg color
```

## Dependencies

Runtime: none (reads from procfs/sysfs).
Build: Go 1.23+, `github.com/charmbracelet/bubbletea v1.3.10`, `github.com/charmbracelet/lipgloss v1.1.0`.
